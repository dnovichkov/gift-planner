# Техническое задание: Приложение управления подарками

## 1. Общее описание

**Название:** Gift Planner (Планировщик подарков)

**Назначение:** Веб-приложение для планирования, управления и отслеживания подарков к праздникам.

**Основная цель:** Упростить процесс подготовки подарков к празднику, позволяя пользователю:
- Создавать праздники
- Управлять списком одариваемых людей/семей
- Назначать подарки каждому получателю
- Отслеживать статус покупки подарков
- Просматривать полный список покупок для магазина

---

## 2. Функциональные требования

### 2.1 Управление праздниками

**F1.1 Создание праздника**
- Пользователь может создать новый праздник
- Параметры праздника:
  - Название (обязательно): "Новый год", "День рождения", "8 марта" и т.д.
  - Дата праздника (обязательно): для напоминания и сортировки
  - Описание (опционально)
  - Максимальный бюджет (опционально)
- Праздник сохраняется в локальное хранилище

**F1.2 Просмотр списка праздников**
- Вывод всех созданных праздников
- Сортировка по дате (ближайшие впереди)
- Отображение основной информации: название, дата, количество одариваемых, статус готовности (%)

**F1.3 Редактирование праздника**
- Изменение названия, даты, описания, бюджета
- Удаление праздника с подтверждением

**F1.4 Выбор активного праздника**
- При клике на праздник он становится активным
- Отображение деталей активного праздника и связанных с ним данных

### 2.2 Управление одариваемыми

**F2.1 Добавление одариваемого**
- Внутри выбранного праздника добавить нового получателя подарка
- Параметры одариваемого:
  - Имя или название (обязательно): "Иван Сидоров", "Семья Петровых"
  - Тип (обязательно): Взрослый / Ребенок / Семья (для фильтрации)
  - Примечание (опционально): особые пожелания, размер одежды и т.д.
  - Бюджет (опционально)
- Уникальный ID для каждого одариваемого

**F2.2 Просмотр списка одариваемых**
- Для активного праздника показать всех получателей подарков
- Для каждого: имя, тип, количество назначенных подарков, статус готовности
- Фильтрация по типу (все, взрослые, дети, семьи)
- Поиск по имени

**F2.3 Редактирование одариваемого**
- Изменение имени, типа, примечания, бюджета
- Удаление с подтверждением

### 2.3 Управление подарками

**F3.1 Добавление подарка**
- Для выбранного одариваемого в активном празднике
- Параметры подарка:
  - Название (обязательно): "Книга", "Духи", "Носки"
  - Описание (опционально): детали, ссылка на товар
  - Приблизительная стоимость в рублях (опционально)
  - Статус покупки (обязательно): "Не куплен" / "Куплен" (по умолчанию "Не куплен")
  - Приоритет (опционально): Низкий / Средний / Высокий
- Уникальный ID для каждого подарка

**F3.2 Просмотр подарков для одариваемого**
- Список всех подарков для выбранного получателя
- Для каждого: название, описание, стоимость, статус, приоритет

**F3.3 Отметить подарок как "куплен"**
- Быстрое переключение статуса (например, чекбокс или кнопка)
- Изменение цвета/стиля купленного подарка (визуальный feedback)
- Обновление статистики

**F3.4 Редактирование подарка**
- Изменение всех параметров
- Удаление подарка

**F3.5 Список покупок (Shopping List)**
- Объединенный список всех "Не куплённых" подарков для активного праздника
- Для каждого подарка показать:
  - Название подарка
  - Кому предназначен (имя одариваемого)
  - Приблизительная стоимость
  - Приоритет
  - Возможность отметить как "куплен" прямо из этого списка
- Сортировка по приоритету, стоимости, имени получателя
- Расчет общей стоимости всех некупленных подарков
- Расчет количества некупленных подарков

### 2.4 Статистика и обзор

**F4.1 Информационная панель (Dashboard) для праздника**
- Название праздника и дата
- Всего одариваемых: N человек
- Всего подарков: M подарков
- Куплено подарков: K из M (процент)
- Общая стоимость всех подарков
- Потрачено денег / Осталось потратить
- Статус выполнения (%) в виде прогресс-бара

**F4.2 Быстрый просмотр статуса**
- Какие одариваемые еще "не готовы" (не все подарки куплены)
- Какие одариваемые уже готовы (все подарки куплены)

## 4. Сценарии использования

### Сценарий 1: Подготовка к Новому году

1. Пользователь открывает приложение
2. Нажимает "Создать праздник" → вводит "Новый год", дату "31.12.2025"
3. Добавляет одариваемых: "Мама", "Папа", "Сестра", "Жена"
4. Для каждого добавляет по 1-3 подарка
5. Видит список покупок (12 подарков на ~15000 рублей)
6. Идет в магазин, покупает подарки
7. По мере покупок отмечает подарки как "куплены"
8. Видит прогресс готовности (когда все готово → 100%)

### Сценарий 2: Планирование дня рождения сына

1. Создает праздник "День рождения Сергея", дата "10.01.2026"
2. Добавляет получателей: "Сергей (сын)", плюс 5-6 одноклассников
3. Для каждого одноклассника добавляет примечание о возрасте
4. Назначает подарки (конструкторы, книги, игры)
5. Эта информация поможет при выборе подарков
6. Отслеживает готовность при приближении даты

## 5. Технические требования

### 5.1 Архитектура

- **Фронтенд:** HTML5 + CSS3 + JavaScript (ванильный JS или фреймворк на выбор разработчика)
- **Хранилище:** localStorage или IndexedDB (встроенное в браузер)
- **Деплой:** статический сайт (может размещаться на GitHub Pages, Vercel, Netlify)

### 5.2 Структура данных

```json
{
  "holidays": [
    {
      "id": "uuid",
      "name": "Новый год",
      "date": "2025-12-31",
      "description": "",
      "budget": 15000,
      "createdAt": "2025-12-14T13:00:00Z",
      "recipients": ["id1", "id2", ...]
    }
  ],
  "recipients": [
    {
      "id": "uuid",
      "holidayId": "uuid",
      "name": "Иван",
      "type": "adult", // adult, child, family
      "note": "",
      "budget": 3000,
      "gifts": ["gift_id1", "gift_id2", ...]
    }
  ],
  "gifts": [
    {
      "id": "uuid",
      "recipientId": "uuid",
      "holidayId": "uuid",
      "name": "Книга",
      "description": "Фантастика",
      "cost": 500,
      "status": "not_bought", // not_bought, bought
      "priority": "high", // low, medium, high
      "createdAt": "2025-12-14T13:00:00Z"
    }
  ]
}
```

### 5.3 API/Методы

Основные функции приложения:
- `createHoliday(data)` → новый праздник
- `getHolidays()` → список всех праздников
- `updateHoliday(id, data)` → редактирование
- `deleteHoliday(id)` → удаление

- `addRecipient(holidayId, data)` → новый одариваемый
- `getRecipients(holidayId)` → список для праздника
- `updateRecipient(id, data)` → редактирование
- `deleteRecipient(id)` → удаление

- `addGift(recipientId, data)` → новый подарок
- `getGifts(recipientId)` → подарки для одариваемого
- `getShoppingList(holidayId)` → все некупленные подарки
- `updateGift(id, data)` → редактирование
- `deleteGift(id)` → удаление
- `toggleGiftStatus(id)` → переключить статус

- `getHolidayStats(holidayId)` → статистика (готовность, стоимость)

## 6. Интерфейс: основные экраны

### Экран 1: Главная (список праздников)
- Кнопка "Новый праздник"
- Список карточек праздников (название, дата, прогресс, количество подарков)
- Клик на карточку → переход в праздник

### Экран 2: Детали праздника
- Заголовок (название, дата)
- Вкладки / разделы:
  - **Одариваемые** - список с добавлением нового
  - **Список покупок** - все некупленные подарки по категориям
  - **Статистика** - общая информация о готовности
- Кнопка "Назад" или хлебные крошки

### Экран 3: Детали одариваемого
- Имя, тип, примечание
- Список подарков для этого человека
- Кнопка добавления подарка
- Для каждого подарка: название, стоимость, статус (чекбокс), приоритет

### Экран 4: Список покупок (Shopping List)
- Все некупленные подарки активного праздника
- Сортировка/фильтрация
- Для каждого: название + кому + стоимость + приоритет + чекбокс
- Сумма всех некупленных подарков
- Когда пользователь идет в магазин, может открыть этот экран и отмечать куплено

## 9. Глоссарий

| Термин | Определение |
|--------|-------------|
| **Праздник** | Событие, к которому готовят подарки (Новый год, день рождения) |
| **Одариваемый** | Человек или семья, которым будем дарить подарки |
| **Подарок** | Конкретный предмет/услуга для одариваемого |
| **Статус подарка** | "Не куплен" или "Куплен" |
| **Список покупок** | Агрегированный список всех некупленных подарков праздника |
| **Приоритет** | Важность подарка (низкий, средний, высокий) |
| **Бюджет** | Максимальная сумма в рублях на праздник/одариваемого |

## 10. Примечания

- Приложение полностью клиентское - не требует сервера
- Пользователь может использовать одновременно несколько браузеров, но данные не будут синхронизированы (это не требуется для MVP)
- Для упрощения, валюта - российский рубль, но структура позволяет легко добавить выбор валюты
- Все сообщения на русском языке
